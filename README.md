# 系统架构设计师练题台

这是一个可以直接上传到 GitHub 的静态练题应用。

- 不依赖飞书文档。
- 不依赖后端服务。
- 题库随代码发布。
- 作答记录保存在浏览器 IndexedDB。
- 可以导出 `ai-diagnosis.json` 给 Codex 做复盘。

## 在线使用

发布到 GitHub Pages 后，可以直接用浏览器打开：

```text
https://zhang-986.github.io/ruankao-architect-practice/
```

## 本地运行

不要直接双击 `index.html`，浏览器会限制本地 JSON 读取。用一个静态服务器打开：

```bash
cd ruankao_practice_app
python3 -m http.server 4173
```

然后访问：

```text
http://127.0.0.1:4173
```

## 上传 GitHub Pages

1. 把 `ruankao_practice_app/` 目录作为一个仓库上传到 GitHub。
2. 进入仓库 Settings -> Pages。
3. Source 选择 `Deploy from a branch`。
4. Branch 选择 `main`，目录选择 `/root`。
5. 等部署完成后访问 GitHub Pages 地址。

## 使用方式

1. 打开练题页，默认进入继续练习，按题库顺序接着上次位置往后做。
2. 选择练习模式：继续练习、记忆复习、章节练习、真题套卷、错题重刷、收藏题、题库浏览。
3. 点任意选项就会直接判题，不需要再点提交。
4. 答案、解析、来源年份/卷、题号、模块会立即显示。
5. 错题会自动进入记忆复习和错题页，连续做对后复习间隔会拉长。
6. 遇到典型题可以点 `收藏本题`，之后在收藏题模式集中重刷。
7. 章节进度会显示每个模块的已做数、正确率、错题数和待复习数，点击模块即可进入章节练习。
8. 右侧答题卡按 75 题分段，所有题都能翻页跳转。
9. 统计页会显示模块正确率和薄弱项。
10. 数据页点击 `下载 ai-diagnosis.json`。
11. 把这个 JSON 发给 Codex，Codex 可以按模块、错因、408 是否需要补、下一周 6-8 小时计划给你复盘。

## 练习模式

| 模式 | 用途 |
|---|---|
| 继续练习 | 默认入口，按题库顺序接着上次位置往后做 |
| 记忆复习 | 做错和到期题优先，默认最多 20 题，可自行调整数量 |
| 章节练习 | 按模块刷题，例如架构、数据库、网络、安全 |
| 真题套卷 | 选择某一年份/批次，按同一套真题练 |
| 错题重刷 | 只看最近仍然做错的题 |
| 收藏题 | 只看手动收藏的重点题 |
| 题库浏览 | 自由按来源、年份、模块、关键词筛题 |

## 数据说明

题库数据在：

```text
data/bank.json
```

它由上级目录的结构化题库生成：

```bash
node ruankao_practice_app/scripts/export_app_data.mjs
```

当前包含：

- 综合知识选择题：1822 道
- 其中真题选择题：1297 道
- 其中模拟选择题：525 道
- 案例分析：78 道
- 论文题：60 道

真题选择题覆盖 `2009年下半年` 到 `2025年下半年` 中可结构化的公开资料。`2009-2018` 的综合知识由 PDF 自动抽取补齐；`2019年下半年` 当前源文件是扫描版，暂未纳入可判题题库，后续需要 OCR 或人工复核。

## 隐私说明

练题记录只保存在你的浏览器 IndexedDB 里。除非你主动导出 JSON，否则不会上传到任何地方。

## 诊断 JSON 内容

`ai-diagnosis.json` 主要包含：

- 题库来源和题量
- 总作答数、正确率、错题数
- 模块正确率
- 收藏题 ID
- 最近错题题干、选项、答案、解析
- 全部作答轨迹
- 给 AI 的诊断提示
